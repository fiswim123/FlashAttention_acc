# ADR-003: 除法器 — 迭代 SRT

## 状态

已接受

## 背景

在线 softmax 归一化需要计算 O[i] = acc / l。除法器是面积和延迟的关键权衡点。

## 决策

选择迭代 SRT (Sweeney-Robertson-Tocher) 除法器。

## 理由

| 方案 | 面积 | 延迟 | 选择 |
|------|------|------|------|
| Newton-Raphson | 大 (乘法器) | 低 (4-6 cycles) | |
| 迭代 SRT | 小 | 中 (16 cycles 固定) | **选择** |
| 组合除法器 | 极大 | 极低 (1 cycle) | |

- 迭代除法器面积小，适合资源受限设计
- 固定 16 cycles 延迟可接受（每行 64 元素共享）
- 固定迭代次数，逻辑最简单，无需配置逻辑

## 实现细节

```
被除数: acc (40-bit)
除数: l (40-bit)
商: quotient (16-bit Q8.8)
迭代次数: 16 (固定)

每行计算 64 个元素共享一个除法器:
总延迟 = 64 * 16 = 1024 cycles
```

## 后果

- 面积: ~10K gates
- 延迟: 每行 1024 cycles (固定)
- 可优化: 流水线化，每 cycle 启动一个新除法

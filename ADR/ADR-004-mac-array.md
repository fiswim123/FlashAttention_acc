# ADR-004: MAC 阵列 — 16-wide 向量

## 状态

已接受

## 背景

Q*K^T 和 score*V 的 MAC 运算是主要计算负载。需要在并行度和面积之间权衡。

## 决策

选择 16-wide 向量 MAC 阵列。

## 理由

| 方案 | 并行度 | 面积 | 控制复杂度 | 选择 |
|------|--------|------|------------|------|
| 脉动阵列 | 高 | 大 | 高 | |
| 向量 MAC (8-wide) | 低 | 小 | 低 | |
| 向量 MAC (16-wide) | 中 | 中 | 低 | **选择** |
| 向量 MAC (32-wide) | 高 | 大 | 中 | |

- 16-wide 并行度适中，每 cycle 16 个 MAC
- 向量架构控制简单，易于实现
- 面积 ~200K gates，可接受

## 实现细节

```
16 个 MAC 单元并行:
- 每个 MAC: 16-bit * 16-bit -> 40-bit
- 流水线: 1 cycle/instruction
- 累加器: 64 个 40-bit

Q*K^T 计算:
- 64 个元素，16-wide 并行
- 需要 4 轮 (64/16)
- 每轮 16 cycles (Bc=16)
- 总计: 64 cycles
```

## 后果

- 面积: ~200K gates
- 吞吐量: 16 MACs/cycle
- 计算延迟: 64 cycles/row

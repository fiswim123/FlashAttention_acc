# FlashAttention 加速器 — 赛题完成情况核查

## 一、基本功能要求（必选）

### (1) 算法定义 ✅
- **要求**: O_i = sum_j softmax(Q_i*K_j/sqrt(d) + M_ij) * V_j
- **状态**: ✅ 已实现，在线 softmax + 分块计算
- **验证**: 198/202 测试通过，functional coverage 100%

### (2) FlashAttention-style 计算约束 ✅
- **要求**: 禁止显式存储 SxS 注意力矩阵
- **状态**: ✅ 无 SxS 存储，仅维护 m/l/acc 状态

- **要求**: 必须使用在线 (online) softmax
- **状态**: ✅ fa_softmax 模块实现在线 softmax

- **要求**: 必须分块 (tiling) 处理 K/V
- **状态**: ✅ Bc=16 分块，fa_ctrl 控制 tile 循环

### (3) 固定输入规模 ✅
- **要求**: S=256, d=64, batch=1, head=1
- **状态**: ✅ 固定参数，RTL 中硬编码

### (4) 数据格式（定点）⚠️ 部分完成
- **要求**: Q/K/V 输入 Q8.8 (16-bit 有符号)
- **状态**: ✅ 已实现

- **要求**: Dot-product 累加至少 32-bit (建议 40-bit)
- **状态**: ✅ 40-bit 累加器

- **要求**: softmax 路径允许更高位宽或分段缩放
- **状态**: ✅ 使用 256-entry LUT + 线性插值

- **要求**: 输出 O Q8.8 (16-bit 有符号)
- **状态**: ✅ fa_divider 输出 Q8.8

### (5) 接口要求 ✅
- **要求**: AXI4-Lite 控制 + AXI4 Master DMA
- **状态**: ✅ fa_regfile (AXI4-Lite) + fa_dma (AXI4 Master)

### (6) 寄存器 ✅
- **要求**: 16 个寄存器，偏移 0x00-0x40
- **状态**: ✅ 全部实现，含 CTRL/STATUS/CFG/地址/STRIDE/NEG_LARGE/SCALE/CYCLES

### (7) 存储与资源约束 ⚠️ 需改进
- **要求**: 禁止存储 score/p 全矩阵
- **状态**: ✅ 无 SxS 存储

- **要求**: 允许缓存一小块 K,V tile
- **状态**: ✅ Bc=16, 片上 ~4KB

- **要求**: 允许每行维护 m/l/acc
- **状态**: ✅ 已实现

### (8) 正确性验收 ⚠️ 需改进
- **要求**: mean_abs_error ≤ 0.03, max_abs_error ≤ 0.10
- **状态**: ⚠️ 端到端精度测试未完成（fa_top 有 4 个 timing 相关测试失败）

## 三、关键问题分析

### 问题 1: 存储器实现 ❌ 严重

**当前设计**:
```
k_tile_buffer: 寄存器实现 (~2KB)
v_tile_buffer: 寄存器实现 (~2KB)
q_buffer: 寄存器实现 (~128B)
o_buffer: 寄存器实现 (~128B)
exp_lut: 寄存器实现 (~512B)
register_file: 寄存器实现 (~2KB)
```

**问题**:
1. 寄存器面积远大于 SRAM（约 6-10 倍）
2. 76K gates 中大部分是寄存器，不是逻辑
3. 赛题要求"含存储器折算"，需要用 SRAM 宏单元

**解决方案**:
1. 使用 ASAP7 SRAM 宏单元（asap7_sram_0p0）
2. 或在报告中明确说明 SRAM 面积折算方法

### 问题 2: 综合流程不完整 ⚠️

**当前问题**:
1. 简化的 liberty 文件（缺少完整时序模型）
2. ABC 优化被跳过
3. 未运行完整 STA（多工艺角）
4. 未使用 Genus（赛题要求）

**影响**:
- 面积估算不准
- 时序分析不完整
- 无法确定实际 Fmax

### 问题 3: 端到端验证未完成 ⚠️

**当前状态**:
- 模块级验证: 198/202 通过
- 端到端验证: fa_top 有 4 个测试失败（timing 相关）
- 精度验证: 未完成（需要端到端仿真）
